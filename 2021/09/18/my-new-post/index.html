复习一下利用Github搭建自己的个人博客。
零基础开始学习如何搭建博客，与其说是自己搭建的不如说是自己作为一个搬运工将每个教程的部分搬运
来自己操作，说是看攻略也不为过。本篇主要参考![]https://www.cnblogs.com/chenlove/p/15058170.html

环境准备：
在搭载属于自己的博客之前，环境准备必不可少，且顺序一定要正确。
1.安装git
官方地址：https://git-scm.com/
安装后打开Git Bash Here
输入git –version 确定是否安装成功
2.安装node.js
官方地址：https://nodejs.org/en/download/
安装完成后在Git Bash中输入 node -v 查看是否安装成功
3.安装hexo
输入安装命令 npm install hexo -g
测试是否安装成功 hexo -v
4.安装hexo依赖
npm install – save hexo-deployer-git
git配置SSH key

配置SSH key可以免密的将本地的源码和资源上传到github，无需要每次都输账号和密码
使用命令 cd~/.ssh查看是否配置SSH key
输入 shh-keygen -t rsa -C”邮箱地址” 后回车 并且输入yes
即可在user-administrator中生成ssh文件夹
进入ssh文件夹，复制id_rsa.pub中的内容（可使用记事本打开），（未带pub属于私人文件）
通过github 中New SSH key 添加key与任意title
使用 ssh -T git@github.com查看是否成功
配置账号和密码
$ git config –global user.name “github用户名”
$ git config –global user.email “github注册邮箱”
新建一个保存博客的目录（并使用GITBASH打开）
这里我因为没有使用gitbash打开保存博客目录，导致一直找不到存放目录出现错误。
使用hexo init初始化博客
hexo g生成静态网页
hexo s浏览网页，可通过 http://localhost:4000/ 访问
一定还要部署一个github的库
New repository 给仓库命名时要用博客名
再编辑刚刚保存博客目录下的_config.yml文件
拖到最下面
改写以下内容
deploy:
type: git
repository: git@github.com:用户名/用户名.github.io.git
branch: main （可能是master也可能是main看看自己建立的什么）
使用hexo d 在浏览器中就可以访问了
网址：https://用户名.github.io/
（仓库名就是github博客的访问地址）
